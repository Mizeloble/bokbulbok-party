# 그로스 · 홍보 로드맵

홍보를 "어디에 글을 뿌릴까"로 시작하지 않는다. 이 제품은 **구조 자체가 바이럴**이다 — 한 판 = 4~12명이 QR을 찍고 화면을 본다. 문제는 바이럴 고리가 **절반만** 구현돼 있다는 것. 채널보다 먼저 그 구멍을 막는 게 어떤 광고보다 leverage가 크다.

> **핵심 통찰**: 매 판마다 11명이 목격하는데 아무도 브랜드를 들고 나가지 않는다. 가장 감정이 터지는 순간("OO님 벌칙 당첨 🎯")이 아무 흔적도 안 남긴다. 한국 모임에서 다음 약속이 잡히는 곳은 그 **단톡방**인데, 거기로 브랜드가 안 들어간다. **목격자(11명)를 다음 호스트로 전환하는 고리**를 완성하는 게 1순위.

## 현재 공유 구조의 구멍

- [`InviteSheet.tsx`](../src/components/InviteSheet.tsx) — `navigator.share`로 *방 URL*을 뿌린다(`shareText: "같이 벌칙 정하기 하실래요?"`). 즉 **"지금 이 방에 들어와"**라는 *판 시작 전* 초대.
- 결과 화면([i18n `result`](../src/lib/i18n.ts) L88~)엔 `다시 하기`만 있고 **결과를 밖으로 가져갈 공유물이 없다**. → *판 종료 후* 고리 부재 = 바이럴 누수의 핵심.
- OG는 [`opengraph-image.tsx`](../src/app/opengraph-image.tsx)에서 한글 폰트(Pretendard) 주입까지 렌더 중 — **렌더 기법은 재사용 가능**하나, 결과 카드는 *방 상태를 읽지 않는다*(아래 주의).

## Tier 0 — 인프라 점검 (홍보 *전에* 필수)

홍보가 성공하면 사이트가 죽는 구조를 먼저 없앤다. 숏폼 하나 터져 동시 50팀이 들어오면 콜드스타트 + 11번째 방부터 503.

- [ ] **VM 수직 확장** — `fly.toml` `shared-cpu-1x`/512MB → `2x`/1GB+ (scale-to-zero라 유휴 비용은 그대로 0). [launch-checklist Phase 2](launch-checklist.md) 참조.
- [ ] **`ROOM.MAX_ROOMS` 상향**(현 10) — 반드시 VM 크기 + `http_service` 동시연결 한도와 함께([`src/lib/constants.ts`](../src/lib/constants.ts) 주석).
- [ ] **레이트리밋 여유 확인**([`RATE_LIMIT`](../src/server/rate-limit.ts)) — 정상 트래픽 스파이크를 429로 막지 않게.

## Tier 1 — 제품: 공유 고리 완성 (최우선 작업)

1. [x] **결과 공유 카드** ⭐ — `src/lib/share-card.ts` + `ResultShareButton`으로 구현됨(방식 A). — 꼴찌 결정 순간 → "OO님이 오늘 벌칙 🎯 / 복불복" 이미지 카드 → 한 탭 공유·저장. **단톡방에 브랜드를 박는 핵심.** 문구는 i18n([`src/lib/i18n.ts`](../src/lib/i18n.ts))에만.
   - ⚠️ **데이터 경로 주의(적대적 검증으로 교정됨)**: 방 상태는 메모리 전용 + 유휴 GC([`server/CLAUDE.md`](../src/server/CLAUDE.md) DB 금지 불변조건)이고 OG는 소셜에 캐시되므로, **서버 OG 라우트가 방 상태를 읽어 카드를 만드는 방식은 불가**. 결과 데이터는 이미 클라이언트에 있으니 둘 중 하나로:
     - **(A) 클라이언트 캔버스 렌더(권장)** — 결과 순간 client에서 이미지 그려 `toBlob` → `navigator.share`. 서버·영속성 불필요.
     - **(B) URL에 결과 인코딩** → 무상태 `next/og` 라우트가 쿼리 파라미터로 렌더(Pretendard 폰트 주입 기법만 재사용, 방 `Map` 미접근).
2. [x] **결과 화면 CTA 강화** — 결과 공유 옆 "친구 부르기" 버튼으로 [`InviteSheet`](../src/components/InviteSheet.tsx) 재사용(v2.12.0).
3. [ ] **외울 수 있는 진입점(도메인)** — `bokbulbok-party.fly.dev`는 입으로 못 옮긴다. 짧은 도메인이 바이럴의 전제. [launch-checklist "나중: 도메인"](launch-checklist.md) 항목과 동일 작업(`SITE_URL`/`ALLOWED_ORIGIN` 교체).

## Tier 2 — 채널 (무료·한국 특화, Tier 1 이후)

- [ ] **숏폼(1순위)** — 마블 레이스 피니시는 그 자체로 마블런/ASMR 장르(조회수 잘 나오는 포맷). "친구들이랑 벌칙 정하는 법" 15초 화면녹화 → Reels/Shorts/TikTok. 제작비 0, 제품이 곧 콘텐츠. [`scripts/capture-demo.mjs`](../scripts/capture-demo.mjs)로 깔끔한 녹화 가능.
- [ ] **개발자 커뮤니티(오픈소스 앵글)** — MIT라 [GeekNews](https://news.hada.io), OKKY, 페북 개발자 그룹에 "혼자 만든 파티게임" 글이 잘 먹힘. 글 쓰는 개발자 = 모임 주최자.
- [ ] **상황 맥락 시딩** — 트리거는 회식·MT·술자리·카페모임. 대학(에브리타임), 오픈채팅(모임·술게임방), 직장(블라인드)에 *자연스럽게*. 스팸은 역효과.
- [ ] **네이버 SEO** — 한국 검색은 네이버. "복불복 게임 / 벌칙 정하기 룰렛" 유입용 텍스트 콘텐츠 + 블로그(구글 SEO보다 우선).

## Tier 3 — 나중 (트래픽이 정당화하면)

- [ ] **애드핏 수익화** — 도메인 확보 후. [launch-checklist 광고망](launch-checklist.md) 절차대로. 유료 *광고 집행*은 LTV가 없어 지금은 무의미(획득은 바이럴·콘텐츠로).
