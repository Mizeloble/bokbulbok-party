# GeekNews 초안 (news.hada.io)

> 채널: GeekNews — "Show GN" 톤. 1순위, 동시 유입 가능 → §4-A(상시 오픈) 권장.
> 게시 전: 배포 머신 깨워두기(상시면 자동), 데모 GIF 현재 UI와 일치 눈으로 확인.

## 제목 (택1)

- `Show GN: 복불복 — 설치 없이 폰으로 같이 하는 벌칙 정하기 파티 게임`
- `Show GN: DB 없이 서버 메모리만으로 만든 실시간 멀티플레이 파티 게임`

## 본문

술자리·모임에서 "누가 벌칙?" 정하는 걸, 앱 설치 없이 폰만 있으면 같이 할 수 있게 만들었습니다.

호스트가 웹에서 방을 열면 QR이 뜨고, 참가자들은 스캔해서 바로 입장합니다(2~30명). 미니게임 한 판으로 누가 벌칙을 받을지 정해지고, 모두의 화면에서 **똑같은 애니메이션이 같은 타이밍에** 재생됩니다.

게임은 6종입니다 — 마블 레이스 / 응원 마블 레이스 / 기울임 마블(자이로) / 동시탭 반응속도 / 일반 상식 퀴즈 / 넌센스 퀴즈.

**구조적으로 재미있었던 점:**

- **서버가 결과의 유일한 권위자.** 클라이언트 입력을 진실로 신뢰하지 않습니다. 서버가 시드 기반으로 결과를 먼저 계산하고, 그걸 "리플레이 트랙"으로 한 번에 브로드캐스트합니다. 모든 클라가 동일한 wall-clock에 같은 트랙을 재생 → 화면이 갈리지 않습니다. 실시간 프레임 스트리밍이 아니라서 30명이 붙어도 대역폭이 입력 크기로 고정됩니다.
- **마블 물리는 결정론적.** box2d-wasm + 시드 RNG로 같은 시드 → 항상 같은 결과. 서버가 한 번 돌려 승패를 확정하고 클라는 그걸 그대로 그립니다(공정성 검증 스크립트 포함).
- **DB가 없습니다.** 방·플레이어 상태는 전부 서버 프로세스 메모리(`Map`). 서버 재시작 = 전부 초기화가 정상 동작입니다. 같은 그룹이 두 번 켤 이유가 없는 일회성 파티 게임이라, 영속 저장이 오히려 그룹 간 격리 문제만 만든다고 봤습니다.
- **공개 운영 가드.** 단일 인스턴스(Fly shared-cpu-1x / 512MB) 전제라 전역 동시 방 수 상한 + 방 생성 IP 레이트리밋 + 소켓 플러드 가드로 OOM/남용을 막습니다. 초과 시 503/429로 우아하게 degrade.

스택: Next.js 16(App Router) + 커스텀 Node 서버 + Socket.IO(동일 포트), TypeScript. 한국어 UI, MIT 오픈소스.
